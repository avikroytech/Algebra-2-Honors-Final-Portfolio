RUNTIME COMPLEXITY

DEFINITION

Runtime complexity, also known as time complexity, measures the computational efficiency of an algorithm by analyzing how the execution time grows relative to the input size.

T(n) = f(n)

Where T(n) represents time as a function of input size n

STORY

This was the first topic I explored for this project and it was the spark that inspired me to explore computer science for my final project. I thought about how different runtime complexities were so similar to the function families and equations we learned about in Algebra 2 that I knew I had to test and discover the mathematics within this topic.

ALGORITHMS

STORY

After exploring how runtime works with programs, I wanted to apply it deeper into computer science. I decided to focus on algorithms, specifically searching and sorting algorithms. I chose known ones to implement using Python and tested them with inputs. I compared the results and data of my tests with predicted measurements, and found very similar outcomes. This really helped show how math really can help me enhance my skills in computer science and proved that runtime complexity is actually true when empirically tested.

DATA STRUCTURES

DATA STRUCTURES

Data structures are specialized formats for organizing, processing, retrieving and storing data. They provide efficient ways to access and modify data based on specific use cases and requirements.

DATA STRUCTURE EXAMPLES

Arrays

Contiguous memory locations storing elements of the same type, providing O(1) random access.

int arr[5] = {1, 2, 3, 4, 5};

[Graph Visualization: Arrays]

Linked Lists

Dynamic data structure with nodes containing data and pointers to next elements.

struct Node { int data; Node* next; };

[Graph Visualization: Linked Lists]

Trees

Hierarchical structure with nodes connected by edges, used for efficient searching and sorting.

struct TreeNode { int val; TreeNode* left, right; };
